# Glints Interview Task

Description
  Build a simple user-facing webapp that allows the user to filter for restaurants open by date time as well as restaurant name. On top of that, users can save restaurants into their own named collections (eg. Vegetarian favourites, Meat-lovers etc.).
  
  More information on task can be found [here](https://gist.github.com/seahyc/d013a8f8f1c1be52513cf7b77cce6e81).

### Components
- _glints-frontend_ - Web application for the task
- _glints-backend_ - API for the task

### Timebox
  As per task description [here](https://gist.github.com/seahyc/550d9302571079b2ca51ea403710048c), task was timeboxed to 48hrs.
  
### Requirements
- [x] Allow the user to filter for restaurants open by date time as well as restaurant name
- [ ] Users can save restaurants into their own named collections (eg. Vegetarian favourites, Meat-lovers etc.)
- [ ] BONUS: Users can also invite their friends to collaborate (add, remove, edit name) on the collections via email, and they can see updates to their collections in real-time, without page refreshes.

### Tech stack

| Frontend      |  Backend           | Database |
| ------------- | ------------------ | -------- |
| React + Redux + Axios  | Spring    | Postgres |
